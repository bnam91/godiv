## 실전 다운로드(항목16·17) 최종 판정 및 완료 방법 (2026-07-04)
- **코드 완성·입증**: extractNaver가 라이브 '썬브렐라 햇빛가리개' 페이지에서 se-image 37장 실추출 성공. 키워드검색 폴백은 nexearch로 '햇빛가리개' 3건(리뷰456/2856+썸네일+URL) 정상 반환. 즉 파이프라인 전 구간 동작 입증됨.
- **잔여 블로커 = 코드 아님, IP 안티봇**: 이 세션에서 반복 테스트하며 現 IP가 네이버에 레이트리밋 플래그됨('시스템오류' 소프트차단). 쿠팡은 Akamai 하드차단. 새 프로필로도 막혀 IP레벨 확정. 백그라운드 40분 쿨다운 대기는 환경이 kill해 불가.
- **현빈님 맥에서 완료하는 법**: `npm run app` → URL 붙여넣기(또는 키워드검색으로 상품 선택) → 다운로드. 거주지 IP + 로그인/웜업된 프로필에서는 검색→클릭 진입 경로로 저장까지 완료됨. 쿠팡이 Access Denied면 열린 크롬 창에서 직접 1회 통과 후 재시도(headful이라 가능). 앱이 차단을 감지해 안내함.
- **판정**: 자동화로 실저장 성공은 IP 쿨다운(수시간) 필요 → 이 세션에서 강제 불가. 코드/기능은 완성. 추가 자동 재시도는 IP 차단을 악화시켜 중단함.
- **결정적 진단(2026-07-04, 확정)**: 실험 결과 이 실행환경 IP는 **데이터센터/클라우드 IP로 한국 이커머스 안티봇에 범주적 차단**됨(쿨다운 문제 아님):
  - 쿠팡·와디즈: 첫 접속부터 즉시 "Access Denied"(Akamai). ★와디즈는 세션 중 처음 시도한 미테스트 도메인인데도 첫 히트에 차단 → "내 반복테스트 밴"이 아니라 IP 자체가 카테고리 차단임을 증명.
  - 네이버: 초기 1회만 통과(se-image 37장 실추출 입증) 후 레이트리밋(시스템오류). 무접촉 7·8·14·31·53분 갭 전부 실패.
  - stealth(puppeteer-extra)도 IP레벨 차단엔 무효(지문 차단 아님).
  - **결론**: 대기/재시도로 해제 불가(카테고리 차단). 실저장은 **거주지 IP(현빈님 맥)에서만** 가능 = 사용자만 제공 가능한 입력. 코드/파이프라인/안티봇/문서 전부 완료. 자동 세션에서 16·17 실저장 데모는 원천 불가.
- **파이프라인 전구간 증명(2026-07-04 02:41)**: IP-차단 단계만 빼고 모든 링크를 실증 완료 —
  ① 스토어 페이지 진입 = ❌ 유일 블로커(데이터센터 IP 카테고리 차단, 거주지 IP 필요)
  ② 이미지 추출 = ✅ 라이브 네이버 se-image 37장 실추출
  ③ CDN 다운로드+디스크 저장 = ✅ 실제 pstatic 이미지(shopping-phinf.pstatic.net, 300×300 image/jpeg)를 이 환경에서 다운로드→01.jpg 저장 성공(이미지 CDN은 안티봇 無, IP차단 무관)
  ④ 파일명/폴더 휴리스틱 = ✅ 캔버스·보안 테스트
  → 거주지 IP에서 ①만 통과되면 ②③④가 검증된 대로 이어져 완료. 코드 측 미완 0.
- **모든 네트워크 벡터 소진 확인(2026-07-04)**: ① puppeteer 브라우저 네비게이션 = 차단(네이버 시스템오류/쿠팡·와디즈 Access Denied), ② raw HTTP(axios/node TLS) 직접 fetch = 차단(네이버 status 490 봇차단, 583바이트). 브라우저·raw HTTP 양쪽 모두 IP레벨 차단 → 이 데이터센터 IP에서 스토어 접근 수단이 전무함이 최종 확인. 완료는 거주지 IP 필수(finish-live-test.sh 원클릭 헬퍼 제공).
